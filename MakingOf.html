Stage 01

Problem → Concept

The project started by identifying a real issue: irregular schedules cause poor sleep and weak focus. SunSync aims to match light to the circadian day — warm to wake/wind-down, cool to focus.

Circadian rhythm Target users Value proposition
Stage 02

Planning

With a clear concept, the next step was planning the design and build process in Figma: user flows, wireframes and UI elements to ensure a consistent, polished prototype that effectively communicates the concept.

Figma User flow Consistency
Stage 03

Prototype Build

Once everyone was aligned on the design, the build phase focused on translating Figma screens into a functional HTML/CSS/JS prototype. The goal was to create a realistic experience that effectively demonstrates the concept and design. Sticking to the planned structure and styles ensured a polished, professional result.

HTML CSS Polish
Stage 04

Iteration & Fixes

Final iteration focused on real-world issues like responsiveness, accessibility, and minor bugs. This stage is crucial for polishing the prototype and ensuring it’s a professional, user-friendly experience that effectively communicates the concept. Iteration is a key part of the design process, allowing for continuous improvement and refinement based
